## ЗАДАЧА 1: Сбор YouTube подписок
**СТАТУС:** Завершено

**ЧТО РАБОТАЕТ:**
1. Подключиться к Chrome через `mcp__Claude_in_Chrome__tabs_context_mcp`
2. Открыть `https://www.youtube.com/feed/channels` — страница подписок пользователя
3. Скроллить страницу вниз, собирая данные каждого канала: название, handle, подписчики
4. Парсить данные через `read_page` или `javascript_tool`
5. Записать в Excel через Python (openpyxl) на компе пользователя

**ЧТО НЕ РАБОТАЕТ:**
- YouTube API без OAuth — нельзя получить подписки юзера
- Bash-песочница — нестабильна, часто не стартует ("Workspace still starting")

**ОШИБКИ КОТОРЫЕ ДОПУСТИЛИ:**
- Пытались использовать bash для создания Excel — песочница зависала
- Слишком быстрый скролл — элементы не успевают загрузиться, нужны паузы

## ЗАДАЧА 2: Сбор Instagram подписок
**СТАТУС:** Завершено

**ЧТО РАБОТАЕТ:**
1. Зайти на Instagram через Chrome пользователя (авторизован)
2. Перейти на страницу подписок пользователя
3. Прокручивать список, собирая: username, имя, подписчики, bio, email, сайт, категория, публикации
4. Для каждого аккаунта — переход на профиль для получения деталей
5. Записать через Python openpyxl

**ЧТО НЕ РАБОТАЕТ:**
- Прямой парсинг API Instagram — блокировки
- Слишком быстрый переход между профилями — Instagram может временно заблокировать

**ОШИБКИ КОТОРЫЕ ДОПУСТИЛИ:**
- Нужны паузы между запросами к профилям (2-3 секунды минимум)
- Если Instagram начинает показывать "Rate limited" — остановиться, подождать или сменить аккаунт

**ФАЙЛЫ:**
- Данные вошли в CRM Excel, лист "Instagram" (~400 аккаунтов, 15 колонок)

## ЗАДАЧА 4: Создание единого CRM Excel
**СТАТУС:** Завершено

**ЧТО РАБОТАЕТ:**
Скрипт `create_crm.py` — Python + openpyxl. Создаёт 8 листов:
1. Instagram (400 строк, 15 колонок)
2. YouTube (95 строк, 8 колонок)
3. Telegram (59 строк, 8 колонок)
4. Приоритет A (52 строки)
5. Кросс-платформа (23 совпадения)
6. С Email (12 контактов)
7. Аутрич — мастер-лист (519 строк, 22 колонки)
8. Статистика

Ключевые функции:
- `build_outreach()` — кросс-платформенное сопоставление через нормализацию хендлов (lowercase, strip @, remove _.-) + сравнение имён
- Приоритет A/B/C на основе: кол-во платформ, релевантность ниши (RELEVANT_ROLES), подписчики, наличие email
- Кликабельные гиперссылки на IG/YT/TG/Email
- Секции-разделители (3 платформы / 2 платформы / 1 платформа)

**ФАЙЛЫ:**
- `D:\CLAUDE Cowork\shikardosteam.github.io\create_crm.py` — основной скрипт
- `D:\ЗАГРУЗКИ\CRM_Shikardos_Team_2026-05-02.xlsx` — результат (также в корне проекта)

**ПРОМПТ ДЛЯ ПОВТОРЕНИЯ:**

## ЗАДАЧА 5: Вкладка "Аутрич" в CRM — единый мастер-лист
**СТАТУС:** Завершено

**ЧТО РАБОТАЕТ:**
Функция `build_outreach()` в create_crm.py:
- Шаг 1: Итерация по IG профилям, сопоставление с YT (по хендлу + имени) и TG (по хендлу)
- Шаг 2: Необработанные YT → сопоставление с TG
- Шаг 3: Необработанные TG (без приватных каналов "+")
- Шаг 4: Расчёт приоритета

Логика приоритета:
- A: 3 платформы, или 2 + релевантный/100K+, или 500K+, или релевантный + 50K+
- B: 2 платформы, или релевантный, или 20K+, или есть email
- C: остальные

RELEVANT_ROLES = {"AI-автоматизатор", "Контент-мейкер", "Affiliate", "Маркетолог"}

Результат: 516 уникальных людей, 120 приоритет A

**22 колонки:**
№, Приоритет, Имя, Платформы, Роли, Категория, Описание, YouTube, YT подписчики, Instagram, IG подписчики, IG публикации, Telegram, TG подписчики, Email, Сайт, Доп. ссылки, Всего подписчиков, Страна, Статус, Дата контакта, Примечания
